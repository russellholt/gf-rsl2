Reference Introduction

This is the new class browser. It is a hypertext reference to the D classes, and should serve as the primary reference documentation. Any printed documentation is likely to be out of date with respect to this online reference.

On the left is a class and concept hierarchy. It's not a list of classes explicitly because, for example, there is no C++ class "Collection". There is, however, a DO_Collection and a DR_Collection. However, there is an RSL class "Collection"; In a sense, the RSL class hierarchy is a design hierarchy, and the C++ classes form the multiple implementation hierarchies. Therefore, there are two sets of documentation for each concept: one for RSL's perspective and one from the D/C++ world. Most of these were generated by autodoc, with the exception of the c++ classes marked in red.

  • Clicking on either or will display the documentation for the indicated concept in this frame.
  • To return to this page, click on the Reference Introduction link at the top of the class list on the left.

Finally, as always, the source code should serve as the final reference.

Visual

The D class hierarchy is shown below from both conceptual and class-based angles. Classes in grey do not exist. italics indicates an abstract class. Notice that no reference class is abstract. Also, each "concept" such as List corresponds to an RSL class, making RSL a "design" layer and C++ an "implementation" layer.

There are more than 86 C++ classes in the D system. Rather than show a hierarchy of or document them all individually, it is more useful to categorize them by concept rather than by individual C++ class since, for example, there are four C++ implementation classes that comprise the Collection concept: each of these classes has separate inheritance trees both up and down and so breaking them up reduces the ability to see global relationships that grouping by concept allows.
