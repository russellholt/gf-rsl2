DCollection

generic interface for a collection of objects

DCollection is

  • DO_Collection - the implementation
  • DR_Collection - the "smart pointer" reference to DO_Collection

class DO_Collection

a public subclass of DO_Object . . . D

Public member functions

virtual DR_void add ( DR_Object o, ref_t = primary_ref ) =0;

add an object to the collection. Must be implemented by subclasses.

virtual size_t size ( );

The number of elements in the collection.

DR_Object deepCopy ( );

create a new object whose instance variables are the deepCopy()ies of each instance variable.

dcompare_t compare ( const DRef& d) const;

Compare collections element-by-element, using their enumerated orders.

DO_Enumerator * elements ( ) =0;

Gives all collections the ability to be enumerated. The enumerator object is used to access each element of the collection sequentially; each DO_Collection subclass will define a private subclass of DO_Enumerator.

This is uses as follows:

	DR_List thelist;

	DR_Enumerator e ( thelist->elements() );
	while(e->hasMoreElements()) {
		... e->nextElement() ...
	}

Note that the enumerator is not dealt with as DO_Enumerator * ! It is the job of DR_Enumerator to manage a pointer to a DO_Enumerator.

BOOLEAN toBoolean ( ) const;

BOOLEAN toBoolean ( ) const;

void init ( );

calls DO_Object::init();

void destroy ( );

calls DO_Object::destroy();

DR_String toString ( );

returns a DR_String in a format very similar to ECI_OStream::collection(const DR_Collection& ), except that this method concatenates invocations of toString() for each of its elements (separated by commas). It is almost eci syntax, with a few exceptions, notably it will neither quote nor perform escape sequence substitution for DString objects (eg '\n' to "\n").Use ECI_OStream when streaming for ECI!

DRef route ( DR_Message m);

Answer standard Collection messages. If no message is matched, return DO_Object::route(m).
